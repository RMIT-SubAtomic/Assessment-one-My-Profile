Project Idea

Overview

This tool is a multi-purpose hardware vulnerability and hard drive backup/repair tool. It has the capability to repair hard drives with corrupt disks through bad sector reassignment and also force sector clipping/trimming[5], running diagnostic tools including full support for S.M.A.R.T checking system, disk imager for backup either to another disk or to an external flash drive, ability to reflash the hard drives firmware, run a virus scan on the drive, RAM diagnostic software. This unit also has an ethernet and wifi connection so the user can connect the device to an internet connection and have access to a massive database of firmware patches and install files, viruses and malware information to keep up to date with the latest threats. The user can use the device as a PXE to load operating System iso’s if your workstation or server if it can not boot from a USB or CD/DVD.

Motivation

With the sophistication of some of the current viruses and malware, threat detection and removal can become an extremely difficult task especially when it comes to a firmware virus on a user’s hard drive like EquationDrug or GrayFish [1]. The motivation in regards to the ability to scan a hard drive independently and then be able to back up and repair it, comes from the fact that I have always been looking for a single tool that incorporates those abilities, because you never know what can go wrong when reloading firmware or scanning a device and removing files, so having the ability to back up the drive with a complete image before doing any operations on it is great. The motivation for most of the features incorporated with this device comes from repairing computer disks and data recovery and the fact that there are so many different tools needed that having a single device that can work as a stand-alone unit would be great.

Description

What's the interface going to be like?

This tool will be a stand alone unit, meaning you will not need access to the original computer to work with the drive that needs checking or backing up. At the moment the plans are to build this using a Raspberry Pi 4 b+ with a 7’ touch screen running Black Arch with all the tools needed pre loaded. To plug the hard drives in will be via USB 3. The interface will be easy to understand and well documented using programs that have a lot of support behind them and an active community.

How it will find infections?

This tool will connect to the internet through its on board ethernet port or through a wireless connection to access a database full of manufacturer firmware files as well as virus/malware repositories that will be stored on a server, It will then perform a full scan of either the disk itself or an image of that disk, checking every file for discrepancies or signatures that most virus / malware leave. It can then dump the firmware from the drives rom and reload it with a copy of the original firmware from the manufacturer.

How will it backup files?

There'll be a couple of options when it comes to backing up. First being a full disk image to an external source, making a temporary image to the device itself to be used to scan for infections quickly or using the network connection to copy the image to a server or a locally connected computer.

Future plans

Would like to see an extra ethernet port set on the device allowing traffic to pass through from one port to the next giving the user the ability to capture all the data that is flowing through their network. Also would like to see the device move away from using the raspberry pi and onto a custom PCB with own applications written in Assembly.

Tools and technologies

The base hardware will be a Raspberry Pi 4 with 4GB RAM [2], this is a small compact credit card computer that has specs of:

  • Broadcom BCM2711, Quad-core Cortex-A72 (ARM v8) 64-bit SoC @ 1.5GHz
  • 4GB LPDDR4-3200 SDRAM (depending on model)
  • 2.4 GHz and 5.0 GHz IEEE 802.11ac wireless, Bluetooth 5.0, BLE
  • Gigabit Ethernet
  • 2 USB 3.0 ports; 2 USB 2.0 ports
  • Raspberry Pi standard 40 pin GPIO header (fully backward compatible with previous boards)
  • 2 × micro-HDMI ports (up to 4kp60 supported)
  • 2-lane MIPI DSI display port
  • 2-lane MIPI CSI camera port
  • 4-pole stereo audio and composite video port
  • H.265 (4kp60 decode), H264 (1080p60 decode, 1080p30 encode)
  • OpenGL ES 3.0 graphics
  • Micro-SD card slot for loading operating system and data storage
  • 5V DC via USB-C connector (minimum 3A*)
  • 5V DC via GPIO header (minimum 3A*)
  • Power over Ethernet (PoE) enabled (requires separate PoE HAT)
  • Operating temperature: 0 – 50 degrees C ambient

To connect the hard drives to the Raspberry Pi will be through the USB to SATA cable or a USB 3.0 SATA dock that can connect more than one drive. Running on the Raspberry Pi 4 b+ will be a Linux distro for Data recovery and system admin tasks called Black Arch, Black Arch comes with over 2300 tools [3] that can be used for data recovery, Disk backups, and imaging as well as setting up a server so a computer can use the Device as a PXE. This setup with the Raspberry Pi uses a 7” touchscreen monitor that will be incorporated into a custom 3d printed case giving it the ability to be used as a stand-alone unit and not need to access any system that might have been infected [4].

Skills Required

I am targeting this device to computer professionals but I would like to be able to have the general home user be able to use it as well. There would be some basic skills needed in using the Linux terminal as well as connecting computer hardware but with a well-written help document, I am sure the home user could learn all the functions.

Outcome

If this project is successful it will cut down time used on threat detection / removal, complex disk repairs, data recovery, and firmware repairs. It will also turn what is normally an expensive endeavour into a cheap and viable option for anyone to implement. The ability to use a stand alone device for all of those actions will be very handy.
